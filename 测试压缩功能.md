# 会话记忆压缩功能测试指南

## 一、启动观察

启动应用后，查看日志中的配置信息：

```
=== 会话记忆压缩配置已加载 ===
最大消息数: 20
Token阈值: 6000
保留最近轮数: 5
最近对话Token上限: 2000
降级保留轮数: 10
分布式锁过期时间: 5秒
分布式锁重试次数: 3
==============================
创建TokenCountChatMemoryCompressor Bean
```

## 二、触发压缩的方式

压缩会在以下情况自动触发：
1. **消息数超过20条**
2. **Token数超过6000**

## 三、观察日志

### 1. 正常添加消息（未触发压缩）
```
会话123无需压缩 - 消息数:15, Token数:3500
```

### 2. 触发压缩
```
🔄 会话123触发压缩 - 消息数:21/20, Token数:5200/6000
📊 压缩统计 - 历史消息:11条, 保留最近:10条
生成摘要 - 处理11条历史消息
摘要生成完成 - 长度:85字符
✅ 压缩完成 - 原21条 → 新11条(摘要1条+最近10条)
✅ 会话123压缩完成 - 压缩后消息数:11, Token数:2800
```

### 3. 分布式锁日志（DEBUG级别）
```
会话123获取锁成功
会话123释放锁
```

## 四、测试步骤

1. 启动应用，确认配置加载成功
2. 向同一个sessionId发送超过20条消息
3. 观察日志中的压缩触发信息
4. 检查压缩前后的消息数和Token数

## 五、日志级别配置

application.yml中已配置：
```yaml
logging:
  level:
    com.shanyangcode.infintechatagent.memory: DEBUG
```

如需更详细的日志，可改为TRACE级别。
