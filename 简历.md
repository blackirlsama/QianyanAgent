2024.11 - 2026.03           ByteCoach｜企业级智能 Agent 系统         后端独立开发

项目描述：基于 Java + Spring Boot + LangChain4j 构建的企业级智能 Agent 系统，通过深度整合 RAG、MCP 协议、分布式记忆与工具调用能力，构建具备感知、记忆、规划、行动四大核心能力的智能体。

技术栈：Spring Boot 3.5.9 | JDK 17 | LangChain4j 1.1.0 | Alibaba Qwen | PgVector | Redis 8.4.0 | Prometheus + Grafana | Docker

• RAG 检索增强生成系统设计与优化
  - 采用 text-embedding-v4 模型对企业私有文档进行语义向量化，存储至 PgVector 向量数据库，实现私有知识的精准检索
  - 实现递归文档切分策略（chunk 500 字符、重叠 150 字符），保证语义完整性，召回准确率提升 10-20%
  - 设计查询预处理器，通过停用词过滤与文本规范化，提升模糊查询召回率 20-30%
  - 实现 Rerank 重排序机制（粗排 20 条 + 精排 5 条），确保最相关结果优先返回
  - 开发 RagAutoReloadJob 定时任务，每 5 分钟自动扫描文档目录并加载新增/修改文档，实现知识库动态更新

• 分布式会话记忆系统架构设计
  - 基于 Redis 构建分布式会话记忆存储，通过 MemoryId 实现多用户、多实例下的会话逻辑隔离
  - 设计并实现记忆压缩策略（TokenCountChatMemoryCompressor）：滑动窗口保留最近 5 轮完整对话（约 2000 tokens），历史对话压缩为简短摘要
  - 消息数超过 20 条时自动触发压缩，降低 Token 消耗成本约 60%，同时保证对话连贯性
  - 结合 Redis TTL 策略实现记忆生命周期管理，支持长程多轮对话的上下文保留

• 智能工具调用与 MCP 协议集成
  - 基于 LangChain4j Function Calling 机制，通过 @Tool 注解定义方法 Schema，实现自然语言到业务逻辑的精准转换
  - 集成 MCP（模型上下文协议）标准化模型与工具的交互方式，支持联网搜索、智能邮件发送、天气查询等多工具调用
  - 实现动态 RAG 库丰富工具，支持用户通过对话指令反向同步新知识至文档与向量库，实现知识库的动态自生长

• 全链路安全防护与可观测性体系
  - 基于 LangChain4j Guardrail 构建输入层与输出层双重防护机制：输入层实现敏感词过滤、Prompt 注入检测，输出层进行合规性审核
  - 集成 Prometheus + Grafana 打造可视化监控体系，采集 Token 消耗量、模型调用频次、检索命中率、工具调用成功率等核心指标
  - 构建 Top 用户与会话排行榜，为系统优化、资源调配与精细化运营提供数据支撑，通过监控面板实时发现异常并缩短故障排查时间

• 流式输出与前端交互优化
  - 支持 qwen-max 流式对话模型，通过 SSE（Server-Sent Events）技术实现前后端流式通信
  - 实现对话结果的实时分段返回，避免长文本回复的等待延迟，用户感知响应速度提升 50%+
