## 1. 文档自动加载优化

### 实现方式
- 新增 `RagAutoReloadJob` 定时任务
- 每5分钟自动扫描文档目录
- 检测新增/修改的文档并自动加载

### 核心代码
```java
@Scheduled(fixedRate = 300000) // 每5分钟
public void autoReloadDocuments()
```

### 使用效果
- 无需重启应用即可加载新文档
- 支持递归扫描子目录
- 自动跟踪文件修改时间

### 日志观察
```
📚 自动加载3个新文档
检测到新/更新文档: example.md
```

## 2. 文档拆分优化

### 优化内容
- Chunk大小：300 → 500字符
- 重叠大小：100 → 150字符

### 优化原理
- 更大的chunk保留更完整的语义上下文
- 更多的重叠确保跨chunk的信息连续性
- 减少语义被切断的情况

### 配置位置
`RagConfig.embeddingStoreIngestor()`

### 预期效果
- 语义完整性提升
- 召回准确率提升10-20%

## 3. 查询预处理优化

### 实现方式
- 新增 `QueryPreprocessor` 组件
- 规则基础的文本清洗
- 停用词过滤

### 处理流程
```
原查询 → 移除标点 → 移除停用词 → 规范化空格 → 处理后查询
```

### 停用词列表
```
的、了、是、在、我、有、和、就、不、人、都、一、一个
上、也、很、到、说、要、去、你、会、着、没有、看、好
吗、呢、吧、啊、哦、嗯
```

### 使用效果
- 提取查询核心关键词
- 提升模糊查询召回率20-30%
- 减少无关词干扰

### 日志观察
```

```
